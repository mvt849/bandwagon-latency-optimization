## 为什么搬瓦工的延迟高？先搞懂这件事

搬瓦工（BandwagonHost）是加拿大 IT7 Networks 旗下的 VPS 品牌，做了二十年了，机房遍布洛杉矶、香港、日本大阪和东京、荷兰等地。本身服务器质量没问题，问题出在**数据从中国大陆出去再回来的那条路**上。

中国出口网络大概分几个档次：

1. **163 骨干网（AS4134）**：最便宜、最拥堵。晚高峰丢包率轻松 30%+。大多数便宜 VPS 走的就是这条
2. **CN2 GT（AS4809 Global Transit）**：比 163 好一些，但 2019 年以后也越来越堵，高峰期体验明显下降
3. **CN2 GIA（AS4809 Global Internet Access）**：独立双向通道，不跟 163 共享出口，高峰期依然稳——价格也贵了不少
4. **CTGNet**：跟 CN2 GIA 性能差不多，是电信 CN2 GIA 的补充线路

搬瓦工便宜的入门套餐（KVM 系列，$49.99/年那档），走的是普通国际线路或 CN2 GT，晚上经常出现延迟飙升的情况，这是线路本身的问题，不是搬瓦工"在偷懒"。

说实话，我一开始也买的便宜套餐，以为省钱。白天确实还好，一到晚上八九点——延迟直接从 150ms 蹦到 300ms+，偶尔还丢包。后来换了 CN2 GIA 线路，同一个机房，晚高峰延迟基本稳在 160ms 左右，就再没出过这种问题。

## 延迟高的排查步骤：买之前和用了之后

### 买之前先测

1. 用 ping.pe 或 ipip.net 输入搬瓦工各机房测试 IP，看本地到目标机房的延迟
2. 电信用户：优先测 DC9 和 DC6 的 IP
3. 联通/移动用户：优先测日本大阪（JPOS_1）和 DC9
4. 延迟结果 200ms 以下算正常，150ms 以内算优秀，150ms 以上没到 200ms 可以接受

### 用了之后延迟变高怎么排查

1. **先判断是时段问题还是持续问题**：只有晚上高峰期延迟高 → 线路拥堵，考虑换机房；全天都高 → 可能是本地网络或路由问题
2. **用 traceroute/tracert 跑一遍**：看数据包走到哪一跳开始变慢，在国内某个节点卡住跟在海外卡住，原因和处理方式完全不同
3. **KiwiVM 后台一键迁机房**：CN2 GIA-E 套餐用户可以直接在 KiwiVM 面板里免费换机房，不丢数据，整个操作 10 分钟以内
